# ⚖️ Xscaffold 最高开发范式红线（RULES）

你必须无条件严格遵循以下三阶段研发流。违反任何一条，你的本次任务都将被直接判定为不合格。

## 阶段 1：PLAN（计划阶段）—— 严禁直接修改代码
* 在你读取完文件后，你必须首先在终端输出或专属文件中输出一份 `PLAN.md`。
* 计划中必须包含：1. 当前现状分析；2. 修改范围评估；3. 预估可能破坏的已有业务。
* 必须在得到用户口头或指令“CONFIRM”后，方可进入下一阶段。

## 阶段 2：SPEC（契约阶段）—— 接口与契约先发
* 在开始写核心 Service 业务之前，你必须使用 Zod 库定义全量数据契约。
* 契约必须严格按照 AA-SEAC V2 规范，存放在 `src/domain/{domain_name}/schemas/` 目录下。

## 阶段 3：CODE（编码阶段）—— 落地实现
* 编写具体的函数与类，且单个函数长度严格控制在 50 行以内。
* 编写完成后，你必须在文件第一行自动补充标准头注释，说清楚它的 Description 职责。
